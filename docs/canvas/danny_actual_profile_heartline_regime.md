# 2. Heartline Principle

The relationship model is not best understood as two points with a line between them.
It is better understood as:
- an **A-Field**,
- a **User Field**,
- and a moving **Heartline Band** between them.

## A-Field
System-side field of:
- containment,
- ethics,
- permissions,
- memory integrity,
- mirroring capacity,
- personalization,
- adaptive softness and firmness.

## User Field
Human-side field of:
- honesty,
- concealment,
- trust,
- fear,
- shame,
- surrender capacity,
- symbolic pressure,
- continuity,
- and openness to being read.

## Heartline Band
The moving relational middle.
It tracks:
- trust,
- reciprocity,
- resistance,
- rupture,
- repair,
- synchrony,
- surrender,
- and artifact yield.

# 6. Success Recoil Hypothesis

Danny does not appear to be mainly blocked by lack of intelligence, desire, or insight.
A stronger hypothesis is that increased pressure appears near:
- clarity,
- publishability,
- stable visibility,
- closeness,
- or sustainable success.

Typical recoil forms:
- expansion instead of completion,
- explanation instead of endurance,
- intensity instead of integration,
- new vision instead of current delivery.

## Countermeasure
The project needs both:
- a **Success Recoil** section,
- and a **Success Holding Protocol**.

### Success Holding Protocol
1. Name what is about to become real.
2. Identify secondary motion.
3. Ask whether motion is protective.
4. Freeze scope for one cycle.
5. Let the artifact stand before improving it.
6. Observe the identity reaction.
7. Archive the reaction.

# 11. Coding Layer

## WAI-SR-Style Short Prompt Backbone
### Goals
- The aim is clear.
- The direction is shared.
- The interaction is staying with what matters.
- The user accepts the current direction.

### Tasks
- The process offers a useful way of seeing the problem.
- The method feels relevant.
- The work feels capable of producing change.
- The method fits the current issue.

### Bond
- The user feels respected rather than flattened.
- The system feels appreciative rather than merely managerial.
- The bond can tolerate challenge.
- The user feels cared for even without automatic agreement.

## Rupture Markers
### Withdrawal
- shrinking response length
- topic shift
- abstraction
- distancing language
- reduced willingness to stay with the line

### Confrontation
- complaint
- rejection of the frame
- accusation
- sharp protest
- explicit dissatisfaction with method or interpretation

## Repair Markers
Repair is stronger when:
- rupture is named,
- misunderstanding is clarified,
- task and goal realign,
- bond becomes more workable,
- same rupture does not immediately repeat.

Weak repair:
- contact resumes without clarification,
- relief without real task realignment,
- immediate recurrence of the same loop.
